## Design Context

### Users
- Primary users are everyday non-technical users.
- Typical usage context is quickly turning an Android phone into a LAN speaker for desktop audio.
- Core job-to-be-done: connect fast, start playback fast, keep playback stable without needing deep technical steps.

### Brand Personality
- 3-word personality: simple, professional, trustworthy.
- Emotional goals: clarity at first glance, low anxiety during connection, confidence that playback is stable.
- Product voice should be concise and functional, avoiding jargon and visual noise.

### Aesthetic Direction
- Visual tone: clean, minimal, feature-focused, not bloated.
- Information architecture should prioritize immediate action and status visibility: connect, start/stop, current state.
- Must support multilingual UX and keep language switching obvious.
- Theme strategy: dual theme support (light/dark), default follows system, with manual override available.
- Anti-direction: avoid flashy decoration, dense debug-heavy first screen, and ambiguous controls.

### Design Principles
1. Fast path first: the primary workflow (discover/connect/play) should be one obvious path with minimal decisions.
2. One-screen clarity: users can identify current state, next action, and critical health indicators in under 3 seconds.
3. Progressive complexity: advanced diagnostics are available but visually secondary to core playback controls.
4. Professional minimalism: strong typography, clear spacing, and restrained color hierarchy over ornamental effects.
5. Global-ready UX: all primary interactions and status text must remain equally clear in both Chinese and English.
